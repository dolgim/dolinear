# 칸반 보드

이슈를 워크플로우 상태별 컬럼으로 시각화하는 보드 뷰. 드래그 앤 드롭으로 상태 변경 및 순서 조정이 가능하다.

> **용어 참고**: UI에서 "상태(Status)"로 표시되는 것은 API에서 `workflowState` 필드에 해당한다. 컬럼 순서는 `workflowState.position`, 카드 순서는 `issue.sortOrder`로 결정된다.

## 개요

Issues 페이지에 리스트/보드 뷰 전환을 추가한다. 보드 뷰는 워크플로우 상태를 컬럼으로, 이슈를 카드로 표시하며, 드래그 앤 드롭으로 상태 변경과 순서 조정을 지원한다.

![메인 보드 레이아웃](wireframe-main-layout.png)

## 범위

### 이번에 구현

- Issues 페이지 헤더에 리스트/보드 뷰 토글
- 워크플로우 상태별 컬럼, `position` 순서 정렬
- 보드 가로 스크롤 (컬럼이 뷰포트 초과 시)
- 이슈 카드 (식별자, 우선순위, 제목, 라벨, 담당자, 마감일)
- 카드 클릭 → 이슈 상세 페이지 이동
- 드래그 앤 드롭: 같은 컬럼 정렬 + 크로스 컬럼 이동
- DnD 시각 피드백 (카드 고스트, 컬럼 하이라이트, 삽입 가이드라인)
- DnD 자동 스크롤: 세로(컬럼 내) + 가로(보드)
- 기존 필터 보드에도 적용, 필터 후에도 빈 컬럼 유지
- 빈 상태 / 필터 결과 없음 안내
- DnD 실패 시 롤백 + 에러 토스트
- 뷰 상태 localStorage 저장

### 후속 이슈

- 컬럼 접기/펼치기 (Collapsed columns)
- 보드에서 직접 이슈 생성 (컬럼 하단 "+ New Issue" 버튼)
- 그룹핑 변경 (우선순위/담당자별 그룹핑)
- WIP 제한 (In Progress 최대 N개)
- 수영 레인 (Swimlanes)
- 서브태스크 기능 — 칸반보드 카드에 서브태스크 진행률 표시도 함께 고려할 것
- 보드 키보드 내비게이션
- 모바일 대응
- 계정 레벨 뷰 설정 저장 (현재 localStorage → DB 기반 사용자 설정으로 이관)

### 전제조건

- `GET /issues` 리스트 API에 이슈별 라벨 정보를 포함하도록 수정 (현재 누락됨). 이 작업이 선행되어야 보드 카드에 라벨을 표시할 수 있다.
- 토스트 알림 시스템 도입 (현재 프로젝트에 토스트 라이브러리 없음). DnD 에러 롤백 피드백에 필요.

## 데이터 로딩

보드 뷰는 모든 워크플로우 상태의 이슈를 동시에 표시해야 한다. 현재 `GET /issues` API는 `pageSize` 최대 100개로 페이지네이션된다.

- 초기 구현에서는 `pageSize: 100`을 그대로 사용한다. 대부분의 팀에서 활성 이슈가 100개를 초과하지 않을 것으로 예상한다.
- 100개 초과 시 일부 컬럼이 불완전하게 표시될 수 있으며, 이는 알려진 제한사항이다.
- 후속으로 보드 전용 데이터 로딩 (무한 스크롤 또는 상한 확대)을 검토한다.

## 레이아웃

![레이아웃 구조](wireframe-layout-annotation.png)

### 고정 영역

- **사이드바** (220px): 기존 그대로. 보드와 독립적으로 항상 표시.
- **헤더**: Issues 제목 + 뷰 토글 버튼. 스크롤에 영향받지 않음.
- **필터 바**: 기존 FilterBar 컴포넌트 재사용. 스크롤에 영향받지 않음.

### 스크롤 영역

보드 영역만 스크롤 가능:
- **가로 스크롤**: 컬럼이 뷰포트 너비를 초과하면 보드 컨테이너가 가로 스크롤된다.
- **세로 스크롤**: 각 컬럼은 독립적으로 세로 스크롤된다. 카드가 컬럼 높이를 초과하면 해당 컬럼 내에서만 스크롤.

### 컬럼

- 각 컬럼의 최소 너비: 260px.
- 모든 컬럼은 동일한 너비를 가진다.
- 빈 컬럼(이슈 0개)도 동일 너비로 표시한다.
- 컬럼 순서는 `workflowState.position`에 의해 결정된다.

### 컬럼 헤더

- 상태 색상 원형 아이콘 (12px, `workflowState.color`)
- 상태 이름 (`workflowState.name`)
- 해당 컬럼의 이슈 개수

## 뷰 전환

### 토글 UI

헤더에 아이콘 버튼 2개 (리스트 아이콘, 보드 아이콘)를 배치한다. 현재 활성 뷰의 버튼이 하이라이트된다.

- 리스트 아이콘: 가로 줄 3개
- 보드 아이콘: 세로 직사각형 2개 (높이가 다른)

### 상태 저장

- 뷰 선택 상태를 `localStorage`에 저장한다.
- 키: `dolinear:view-preference:<teamId>` (팀별 독립 저장, `teamId`는 UUID)
- 값: `"list"` 또는 `"board"`
- 페이지 진입 시 저장된 값을 읽어 해당 뷰로 렌더링한다.
- 저장된 값이 없거나 유효하지 않으면 `"list"` (리스트 뷰)를 기본값으로 사용한다.
- 토글 클릭 시 즉시 저장한다 (디바운스 없음).

## 이슈 카드

![카드 상세](wireframe-card-closeup.png)

### 구조

카드는 세로 레이아웃으로 3개 영역으로 구성된다:

1. **상단**: 식별자 (`ENG-10`) + 우선순위 아이콘
2. **중앙**: 이슈 제목 (최대 2줄, 2번째 줄 끝에 말줄임표(…) 표시 via `line-clamp: 2`)
3. **하단**: 라벨 뱃지 (좌측) + 마감일 (중간, 선택적) + 담당자 아바타 (우측 정렬)

### 요소별 규칙

**우선순위 아이콘**: 막대 차트 스타일.
- Urgent (1): 빨간색, 3개 막대 모두 불투명
- High (2): 주황색, 2개 막대 불투명
- Medium (3): 노란색, 2개 막대 불투명
- Low (4): 회색, 1개 막대 불투명
- None (0): 아이콘 표시 안 함

**라벨 뱃지**: `Label.color` 기반 반투명 배경 + 텍스트 색상. 여러 개일 경우 줄 바꿈(flex-wrap).

**마감일**: 값이 있을 때만 표시. 지난 날짜는 빨간색으로 표시 (overdue). 공간이 없으면 축소됨 (flex-shrink 가능 요소가 아닌 라벨과 아바타 사이에 위치).

**담당자 아바타**: 20px 원형, 이니셜 표시. 미할당 시 숨김.

**상태 아이콘**: 카드에 표시하지 않는다. 컬럼 자체가 상태를 나타내므로 중복.

### 완료된 이슈

Done 또는 Canceled 컬럼의 카드는 **opacity 60%**로 표시하여 시각적으로 구분한다. 단, 드래그 중인 고스트 카드는 출발 컬럼과 무관하게 항상 95% opacity로 표시한다. 소스 플레이스홀더(점선 테두리)는 원래 컬럼의 opacity를 유지한다 (Done/Canceled에서 드래그 시 60%).

### 카드 인터랙션

**호버**: 카드 테두리 밝기 증가 (`rgba(255,255,255,0.15)`).

**클릭**: 이슈 상세 페이지로 이동 (`/workspace/$workspaceSlug/team/$teamIdentifier/issue/$issueIdentifier`). 카드 전체가 클릭 영역이다.

**클릭 vs 드래그 구분**: 포인터가 5px 이상 이동하면 드래그로 판정하고, 그 미만이면 클릭으로 판정한다 (기존 리스트 뷰의 `PointerSensor` activationConstraint와 동일).

## 드래그 앤 드롭

![DnD 인터랙션](wireframe-dnd-interaction.png)

### DnD 공통

모든 DnD 동작에 공유되는 시각 피드백:

- **고스트 카드**: 드래그 중인 카드의 복제본. 약간 회전 (-2deg), 그림자 (box-shadow), 주황색 테두리 글로우, 95% opacity.
- **소스 자리**: 카드가 있던 위치에 점선 테두리 플레이스홀더 표시.
- **삽입 가이드라인**: 드롭 위치에 주황색 가로선 + 원형 끝점 표시.
- **비관련 컬럼**: 드래그 중 소스/타겟이 아닌 컬럼은 투명도를 낮춰 시각적으로 디밍.
- **드래그 취소**: Escape 키 또는 보드 영역 외부(사이드바, 헤더 등)에서 릴리스 → 카드가 원래 위치로 애니메이션과 함께 복귀.

### 같은 컬럼 내 순서 변경

카드를 같은 컬럼 내에서 위/아래로 드래그하여 순서를 변경한다.

- `sortOrder`만 변경 (`workflowStateId`는 유지).
- 새 `sortOrder` = 삽입 위치 양쪽 카드의 `sortOrder` 중간값.
- 맨 위에 놓을 경우: 첫 번째 카드의 `sortOrder - 1`.
- 맨 아래에 놓을 경우: 마지막 카드의 `sortOrder + 1`.

### 크로스 컬럼 이동 (상태 변경)

카드를 다른 컬럼으로 드래그하여 상태를 변경한다.

- `workflowStateId` + `sortOrder` 모두 변경.
- 모든 컬럼이 드롭 타겟이 될 수 있다 (Done, Canceled 포함 — 제한 없음).
- 타겟 컬럼 전체에 주황색 배경 틴트 + 테두리로 드롭 존 표시. Done/Canceled 컬럼도 동일한 하이라이트 적용.
- 삽입 위치의 `sortOrder` 계산은 같은 컬럼 내 순서 변경과 동일한 규칙.

### 빈 컬럼에 드롭

빈 컬럼에 카드를 드롭하는 것은 크로스 컬럼 이동의 특수 케이스이다.

- 첫 번째 카드의 `sortOrder`는 기존 로직을 따른다: 컬럼에 카드가 없으므로 `sortOrder = 0`.
- 이후 같은 컬럼에 추가 카드를 드롭하면, 이미 카드가 존재하므로 일반 삽입 규칙 (중간값/±1)이 적용된다.
- 동일한 컬럼 하이라이트 피드백 적용.

### 자동 스크롤

드래그 중 카드가 스크롤 영역의 가장자리에 접근하면 자동 스크롤을 시작한다.

**세로 자동 스크롤** (컬럼 내):
- 컬럼의 상단/하단 40px 영역에 카드가 진입하면 활성화.
- 가장자리에 가까울수록 스크롤 속도 증가 (선형 보간).

**가로 자동 스크롤** (보드):
- 보드 컨테이너의 좌/우 60px 영역에 카드가 진입하면 활성화 (컬럼 너비가 260px이므로 세로보다 넓은 트리거 영역이 필요).
- 동일한 속도 증가 규칙 적용.

자동 스크롤 중에도 삽입 가이드라인은 포인터 위치에 따라 실시간으로 업데이트된다.

> note: dnd-kit 내장 `autoScroll`을 먼저 테스트한다. 가로+세로 동시 스크롤이 불안정할 경우 `onDragMove` + `requestAnimationFrame` 기반 커스텀 구현으로 대체.

> note: 보드의 DnD는 기존 리스트 뷰 DnD와 별도 구현이다. 리스트 뷰는 단일 컨테이너 내 정렬만 지원하지만, 보드는 멀티 컨테이너 크로스 이동이 필요하므로 `onDragOver` + `DragOverlay` + 멀티 컨테이너 collision detection 등 다른 아키텍처가 필요하다. `sortOrder` 계산 로직과 낙관적 업데이트 패턴은 재사용 가능.

### 낙관적 업데이트와 롤백

1. 드롭 즉시 UI를 업데이트한다 (낙관적 업데이트).
2. PATCH API 호출을 보낸다.
3. API 성공 시: 추가 쿼리 무효화를 하지 않는다 (이미 UI가 최신 상태). 기존 `useUpdateIssue`의 자동 무효화는 보드에서 스킵하여 카드 깜빡임을 방지한다.
4. API 실패 시:
   - 이슈 리스트 쿼리 캐시 (`issues` 키)를 무효화하여 서버 상태로 롤백. 필터 적용 여부와 무관하게 전체 이슈 리스트를 다시 가져온다.
   - 에러 토스트를 표시한다.

### 에러 토스트

- 위치: 화면 우하단.
- 스타일: 어두운 빨간 배경, 빨간 테두리, 에러 아이콘.
- 내용: "Failed to move issue" + 이슈 식별자 + "원래 위치로 복원되었습니다" 메시지.
- 5초 후 자동 닫힘.

## 필터

기존 리스트 뷰의 `FilterBar` 컴포넌트를 그대로 재사용한다.

- 지원 필터: Status, Priority, Assignee, Label.
- 필터가 적용되면 매칭되는 이슈만 카드로 표시.
- **필터 후 빈 컬럼도 유지한다** (DnD 드롭 타겟으로 사용되어야 하므로). 이슈가 원래 없는 빈 컬럼과 필터에 의해 비어진 컬럼의 UI는 동일하다 (의도적).

## 빈 상태

![상태 변형](wireframe-states.png)

### 이슈가 없을 때

- 보드 중앙에 빈 상태 메시지 표시.
- 보드 아이콘 + "No issues yet" + "Create your first issue to get started with the board." 설명.
- "Create Issue" 버튼 (기존 이슈 생성 다이얼로그 트리거). **강조(primary) 스타일** — 이슈 생성은 빈 보드에서의 주요 행동 유도이므로.
- 컬럼 헤더는 계속 표시 (보드 구조 유지).

### 필터 결과 없을 때

- 보드 중앙에 빈 상태 메시지 표시.
- 검색 아이콘 + "No matching issues" + "No issues match the current filters." 설명.
- "Clear Filters" 버튼 (outlined 스타일, 비강조).
- 적용된 필터 칩이 필터 바에 표시된 상태.
- 컬럼 헤더는 계속 표시.
